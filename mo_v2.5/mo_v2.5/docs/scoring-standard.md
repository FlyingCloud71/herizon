# MellowWay 慢游推荐打分标准

本文档说明当前 Demo 的慢游可行性评分规则。

评分逻辑对应代码：

```text
utils/evaluator.js
```

## 评分目标

MellowWay 的打分不是判断一个目的地“好不好玩”，而是判断：

```text
在妈妈当前身体状态下，这个目的地今天适不适合安心慢游。
```

因此，评分更关注：

- 走路会不会超量
- 台阶和坡道是否有压力
- 厕所是否方便
- 能不能随时坐下休息
- 是否暴晒或户外时间过长
- 人流是否容易造成疲劳或紧张
- 轮椅/助行器是否需要额外确认

## 总分规则

初始分为：

```text
100 分
```

系统根据用户填写的妈妈状态和目的地字段逐项扣分。

最终分数会限制在：

```text
20-100 分
```

即使扣分超过 80 分，最低也显示为 `20` 分。

## 推荐等级

| 分数范围 | 推荐等级 | 页面语气 |
| --- | --- | --- |
| 68-100 | 推荐 | 可以作为慢游候选 |
| 48-67 | 谨慎推荐 | 可以去，但需要提前处理风险 |
| 20-47 | 不建议 | 不建议作为当天主要目的地 |

## 参与打分的目的地字段

| 字段 | 含义 | 数值方向 |
| --- | --- | --- |
| `walkMinutes` | 慢游预计步行时间，单位为分钟 | 越高越累 |
| `steps` | 台阶、坡道、上下楼、路面连续性压力，范围 1-5 | 越高压力越大 |
| `restroom` | 厕所数量，单位为个，结合游玩时长计算密度 | 密度越高越方便 |
| `seating` | 坐下休息便利度，范围 1-5 | 越高越方便 |
| `outdoorExposure` | 户外暴露度，范围 1-5 | 越高越容易晒/热/累 |
| `crowdRisk` | 人流、排队、拥挤、噪音风险，范围 1-5 | 越高风险越大 |
| `nursingRoom` | 育婴室、亲子卫生间数量，单位为个 | 越高越方便 |

## 参与打分的用户状态

| 字段 | 页面含义 |
| --- | --- |
| `maxWalkMinutes` | 每日步行上限 |
| `avoidSteps` | 尽量不走台阶 |
| `needFrequentRestroom` | 需要频繁上厕所 |
| `needFrequentRest` | 需要频繁休息 |
| `heatSensitive` | 怕热或怕晒 |
| `avoidCrowd` | 避开人流 |
| `needWheelchair` | 使用轮椅或助行器 |
| `needNursingRoom` | 需要育婴室 |

## 具体扣分规则

### 1. 步行时间

比较：

```text
destination.walkMinutes
profile.maxWalkMinutes
```

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| `walkMinutes <= maxWalkMinutes` | 0 | 步行时间在可控范围内 |
| 超出 1-24 分钟 | -14 | 常规游览会超过妈妈舒适步行上限 |
| 超出 25 分钟及以上 | -22 | 常规游览会超过妈妈舒适步行上限 |

### 2. 台阶和坡道

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“尽量不走台阶”，且 `steps >= 3` | -18 | 台阶或坡道压力偏高，需要减少步行范围 |
| `steps <= 1` | 0 | 台阶压力较低 |

### 3. 厕所数量

厕所密度计算：

```text
restroomPerHour = restroom / (walkMinutes / 60)
```

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“需要频繁上厕所”，且 `restroomPerHour < 3` | -10 | 厕所密度偏低，需要提前安排厕所节点 |
| `restroomPerHour >= 5` | 0 | 厕所密度相对友好 |

### 4. 休息点便利度

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“需要频繁休息”，且 `seating < 4` | -10 | 可坐下休息的地方不够密集 |
| `seating >= 4` | 0 | 沿途有较多可休息点 |

### 5. 户外暴露度

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“怕热或怕晒”，且 `outdoorExposure >= 4` | -10 | 户外暴露时间较长，遇到高温或暴晒会明显增加疲劳 |

### 6. 人流风险

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“避开人流”，且 `crowdRisk >= 4` | -10 | 热门时段人流可能让妈妈紧张或疲劳 |

### 7. 轮椅/助行器

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“轮椅/助行器”，且 `steps >= 3` | -16 | 轮椅或助行器通行需要额外确认 |

注意：这一项会和“尽量不走台阶”叠加扣分。也就是说，当 `steps >= 3`，并且用户同时选择“尽量不走台阶”和“轮椅/助行器”时，台阶相关最多会扣：

```text
18 + 16 = 34 分
```

### 8. 育婴室

| 条件 | 扣分 | 页面提示 |
| --- | ---: | --- |
| 用户选择“育婴室更方便”，且 `nursingRoom <= 0` | -8 | 没有明确育婴室，需要提前确认 |
| 用户选择“育婴室更方便”，且 `nursingRoom = 1` | 0 | 有基础育婴室，建议提前确认位置 |
| 用户选择“育婴室更方便”，且 `nursingRoom >= 2` | +6 | 育婴室数量更友好 |

## 当前不参与打分的字段

以下字段目前只用于展示或数据说明，不直接参与推荐分：

| 字段 | 当前用途 |
| --- | --- |
| `transfer` | 记录接驳便利度，后续可纳入评分 |
| `halfDay` | 标记是否适合半日慢游 |
| `bestTime` | 展示建议出发时间 |
| `dropoff` | 展示建议下车点 |
| `restStops` | 展示可休息地点 |
| `restroomTips` | 展示厕所提醒 |
| `backup` | 展示备用方案 |
| `caveat` | 展示注意事项 |
| `dataQuality` | 说明数据可靠程度 |

## 示例

假设妈妈状态为：

```js
{
  maxWalkMinutes: 30,
  avoidSteps: true,
  needFrequentRestroom: true,
  needFrequentRest: true,
  heatSensitive: true,
  avoidCrowd: true,
  needWheelchair: false,
  needNursingRoom: true
}
```

某目的地字段为：

```js
{
  walkMinutes: 45,
  steps: 4,
  restroom: 2,
  seating: 2,
  outdoorExposure: 3,
  crowdRisk: 5,
  nursingRoom: 1
}
```

扣分过程：

| 项目 | 扣分 |
| --- | ---: |
| 步行时间超出 15 分钟 | -14 |
| 避台阶且 `steps >= 3` | -18 |
| 需要厕所且 `restroomPerHour = 2.7 < 3` | -10 |
| 需要休息且 `seating < 4` | -10 |
| 怕热但 `outdoorExposure < 4` | 0 |
| 避人流且 `crowdRisk >= 4` | -10 |
| 需要育婴室且 `nursingRoom = 1` | 0 |

最终分数：

```text
100 - 14 - 18 - 10 - 10 - 10 = 38
```

推荐等级：

```text
不建议
```

## 后续可优化方向

当前评分是 Demo 规则，适合展示产品逻辑。后续如果要更接近真实可用产品，可以继续加入：

- `transfer` 接驳便利度
- 当日天气
- 实时客流
- 厕所实际距离
- 无障碍通行数据
- 数据可信度权重
- 医疗点/服务台距离
